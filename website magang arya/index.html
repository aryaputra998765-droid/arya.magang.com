Debian Linux

Konfigurasi server-server penting di Debian Linux: DHCP, DNS, SSH, dan Web Server Apache2.

🖧
DHCP Server

DHCP (Dynamic Host Configuration Protocol) Server secara otomatis memberikan konfigurasi jaringan ke client — IP address, subnet mask, gateway, dan DNS server.

📋 Langkah Instalasi DHCP
01Update repository dan install isc-dhcp-server.
📸Install DHCP
STEP 01
02Konfigurasi interface di /etc/default/isc-dhcp-server.
📸Konfigurasi Interface
STEP 02
03Edit /etc/dhcp/dhcpd.conf, isi subnet, range IP, gateway, DNS.
📸dhcpd.conf
STEP 03
04Restart service dan cek statusnya dengan systemctl.
📸Restart Service
STEP 04
05Verifikasi client mendapat IP otomatis dari range yang ditentukan.
📸Verifikasi Client
STEP 05
✅ Verifikasi: Cek IP client dengan perintah ip addr di terminal client Linux.
🔍
DNS Server

DNS (Domain Name System) Server seperti "buku telepon" internet — menerjemahkan nama domain (google.com) menjadi IP address yang bisa dimengerti komputer.

📋 Langkah Instalasi BIND9
01Install BIND9: apt-get install bind9 bind9utils.
📸Install BIND9
STEP 01
02Edit /etc/bind/named.conf.local untuk mendaftarkan zone domain lokal.
📸Zone Config
STEP 02
03Buat file zone db.pkl.local dengan SOA, NS, dan A record.
📸Zone File
STEP 03
04Restart BIND9 dan test dengan nslookup dari client.
📸Test nslookup
STEP 04
💡 Tip: Gunakan named-checkconf dan named-checkzone untuk memvalidasi konfigurasi sebelum restart.
🔐
SSH Server

SSH (Secure Shell) memungkinkan kita mengakses dan mengelola server secara remote dengan koneksi terenkripsi, aman dari penyadapan.

📋 Langkah Konfigurasi SSH
01Install OpenSSH Server: apt-get install openssh-server.
📸Install SSH
STEP 01
02Edit /etc/ssh/sshd_config, atur Port, PermitRootLogin.
📸sshd_config
STEP 02
03Restart SSH dan enable agar otomatis berjalan saat boot.
📸Restart SSH
STEP 03
04Dari client, ketik: ssh user@192.168.100.1 lalu masukkan password.
📸Koneksi SSH
STEP 04
⚠️ Keamanan: Untuk produksi, gunakan SSH Key Authentication dan ubah port default 22 untuk menghindari brute force.
🌍
Web Server Apache2

Web Server menerima permintaan HTTP dari browser client dan mengirimkan halaman web sebagai respons. Apache2 adalah web server paling populer di dunia Linux.

📋 Langkah Konfigurasi Apache2
01Install Apache2: apt-get install apache2.
📸Install Apache2
STEP 01
02Akses http://192.168.100.1 untuk verifikasi Apache2 berjalan.
📸Apache Default Page
STEP 02
03Buat file konfigurasi virtual host di sites-available/pkl.conf.
📸Virtual Host
STEP 03
04Aktifkan site dengan a2ensite pkl.conf kemudian restart Apache2.
📸a2ensite
STEP 04
05Buat folder dan file HTML di /var/www/pkl/ lalu akses dari browser client.
📸Website Live
STEP 05
💡 Tip: Setelah membuat virtual host, tambahkan entri di DNS Server (BIND9) agar domain pkl.local bisa diakses dari semua client.

Cisco Packet Tracer

Simulasi jaringan topologi star menggunakan Cisco Packet Tracer — semua device terhubung ke satu switch pusat.

⭐ Apa itu Topologi Star?

Topologi Star adalah topologi jaringan paling umum. Semua perangkat terhubung langsung ke satu switch pusat. Jika satu kabel putus, hanya satu device yang terpengaruh — jaringan lain tetap berjalan.

Simulasi Topologi Star — Semua PC terhubung ke Switch pusat

📋 Langkah di Packet Tracer
01Buka Cisco Packet Tracer, seret 1 buah Switch (2960) ke area kerja.
📸Buka Packet Tracer
STEP 01
02Tambahkan beberapa PC dan susun melingkari switch.
📸Tambah PC
STEP 02
03Gunakan kabel Copper Straight-Through untuk menghubungkan tiap PC ke switch.
📸Pasang Kabel
STEP 03
04Konfigurasi IP address tiap PC: Desktop → IP Configuration, isi 192.168.1.x.
📸Setting IP PC
STEP 04
05Test koneksi: ping ke IP PC lain dari Command Prompt. Harus berhasil!
📸Ping Test
STEP 05
💡 Catatan: Indikator lampu pada port switch akan berubah dari oranye menjadi hijau jika koneksi berhasil.
