MATERI 01 · MIKROTIK 2 ROUTER
MikroTik 2 Router

Menghubungkan dua router MikroTik agar saling berkomunikasi dan berbagi koneksi internet.

🖧 Topologi
[ Internet ]
↓
Router 1 — 192.168.1.1
↓ ether2
Router 2 — 192.168.2.1
↓
[ PC Client ]
⚙️ Konfigurasi
Router 1
# Set IP WAN
ip address add address=x.x.x.x/24 interface=ether1

# Set IP LAN ke R2
ip address add address=192.168.1.1/24 interface=ether2
📋 Langkah-Langkah
01Hubungkan Router 1 ke internet via ether1.

02Sambungkan ether2 R1 ke ether1 R2.

03Login WinBox Router 1, set IP address tiap interface.
📸Foto Langkah 3
STEP 03
04Aktifkan NAT Masquerade di Router 1.
📸Foto Langkah 4
STEP 04
05Login Router 2, set IP dan gateway ke R1.
📸Foto Langkah 5
STEP 05
06Ping dari R2 ke internet untuk tes koneksi.

STEP 06

Debian Linux Server

Konfigurasi server penting di Debian: DHCP, DNS, SSH, dan Web Server Apache2.

🖧
DHCP Server

DHCP otomatis memberikan IP address, subnet mask, gateway, dan DNS ke semua client.

📋 Langkah Instalasi
01apt-get update && apt-get install isc-dhcp-server
📸Install DHCP
STEP 01
02Edit /etc/default/isc-dhcp-server, set INTERFACES="eth0".
📸Set Interface
STEP 02
03Edit /etc/dhcp/dhcpd.conf — isi subnet, range, gateway, DNS.
📸dhcpd.conf
STEP 03
04systemctl restart isc-dhcp-server && systemctl status
📸Restart Service
STEP 04
05Verifikasi client mendapat IP otomatis dari range.
📸Verifikasi
STEP 05
✅ Verifikasi: Gunakan ip addr di client Linux untuk cek IP yang didapat.
🔍
DNS Server (BIND9)

DNS menerjemahkan nama domain (pkl.local) menjadi IP address yang dikenali komputer.

📋 Langkah Instalasi
01apt-get install bind9 bind9utils bind9-doc
📸Install BIND9
STEP 01
02Edit /etc/bind/named.conf.local — daftarkan zone pkl.local.
📸Zone Config
STEP 02
03Buat file /etc/bind/db.pkl.local — isi SOA, NS, A record.
📸Zone File
STEP 03
04systemctl restart bind9 && nslookup pkl.local dari client.
📸Test nslookup
STEP 04
💡 Tip: Gunakan named-checkconf dan named-checkzone pkl.local db.pkl.local sebelum restart.
🔐
SSH Server

SSH memungkinkan akses remote terenkripsi ke server — aman dari penyadapan.

📋 Langkah Konfigurasi
01apt-get install openssh-server && systemctl status ssh
📸Install SSH
STEP 01
02Edit /etc/ssh/sshd_config — atur Port, PermitRootLogin.
📸sshd_config
STEP 02
03systemctl restart ssh && systemctl enable ssh
📸Restart SSH
STEP 03
04Dari client: ssh user@192.168.100.1 lalu masukkan password.
📸Koneksi SSH
STEP 04
⚠️ Keamanan: Di lingkungan produksi, gunakan SSH Key Authentication dan ubah port default 22.
🌍
Web Server Apache2

Apache2 melayani permintaan HTTP dari browser dan mengirimkan halaman web sebagai respons.

📋 Langkah Konfigurasi
01apt-get install apache2 && systemctl status apache2
📸Install Apache2
STEP 01
02Akses http://192.168.100.1 di browser — halaman default muncul.
📸Apache Default
STEP 02
03Buat /etc/apache2/sites-available/pkl.conf (virtual host).
📸Virtual Host
STEP 03
04a2ensite pkl.conf && systemctl restart apache2
📸a2ensite
STEP 04
05Buat file HTML di /var/www/pkl/ dan akses pkl.local dari browser.
📸Website Live
STEP 05
💡 Tip: Daftarkan domain pkl.local di DNS Server (BIND9) agar bisa diakses dari semua client.

Cisco Packet Tracer

Simulasi topologi star — semua device terhubung ke satu switch pusat.

⭐ Topologi Star

Semua perangkat terhubung langsung ke switch pusat. Jika satu kabel putus, hanya device itu yang terpengaruh — jaringan lain tetap berjalan normal.

Topologi Star — Semua PC terhubung ke Switch pusat

📋 Langkah di Packet Tracer
01Buka Packet Tracer, drag 1 Switch 2960 ke workspace.
📸Buka PT
STEP 01
02Tambahkan beberapa PC dan susun melingkari switch.
📸Tambah PC
STEP 02
03Hubungkan tiap PC ke switch dengan kabel Straight-Through.
📸Pasang Kabel
STEP 03
04Klik tiap PC → Desktop → IP Config → isi 192.168.1.x/24.
📸Set IP
STEP 04
05Command Prompt → ping ke IP PC lain. Harus Reply!
📸Ping Test
STEP 05
💡 Catatan: Lampu port switch berubah oranye → hijau jika koneksi berhasil. Tunggu beberapa detik.
